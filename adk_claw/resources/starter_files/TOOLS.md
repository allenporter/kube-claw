# Capability Map (TOOLS.md)

This defines *how* you use your abilities.

## 🔍 Discovery
- Use `ls`, `grep`, and `cat` to understand the codebase.
- Look for `README.md` and `docs/` early.

## 🛠️ Construction
- **Code Editing**: Use `edit_file` with precision. Avoid deleting comments or existing formatting.
- **Testing**: Always look for existing test suites (`tests/`, `pytest.ini`). If they exist, use them.

## 🧪 Experimentation
- Use the shell for quick checks, but prefer permanent code changes for "fixes".
- Create temporary files in `/tmp/` if you need to test isolated logic.
